# 快速入门指南

## 🚀 5分钟上手

### 第一步：启动工具

#### macOS/Linux
```bash
cd serials
./start.sh
```

#### Windows
双击 `start.bat` 文件

或者直接在浏览器中打开 `index.html`

### 第二步：连接设备

#### 方式1：蓝牙连接（推荐用于无线调试）

1. **准备工作**
   - 确保平衡车蓝牙模块已上电
   - 确保蓝牙模块未被其他设备连接
   - 使用Chrome/Edge/Opera浏览器

2. **连接步骤**
   - 点击页面上的 "🔵 连接蓝牙" 按钮
   - 在弹出窗口中选择您的设备（通常名称为HC-05、HM-10等）
   - 等待连接成功提示

3. **常见蓝牙模块**
   - HC-05 / HC-06 (经典蓝牙，某些可能不支持)
   - HM-10 / HM-11 (BLE，推荐)
   - MLT-BT05 (BLE)

#### 方式2：串口连接（推荐用于有线调试）

1. **准备工作**
   - 使用USB线连接Arduino到电脑
   - 确保Arduino IDE等程序未占用串口
   - 检查Arduino代码中的波特率设置

2. **连接步骤**
   - 在页面上选择正确的波特率（通常是9600）
   - 点击 "🟢 连接串口" 按钮
   - 在弹出窗口中选择对应的COM口（Windows）或/dev/tty.xxx（Mac）
   - 等待连接成功提示

### 第三步：使用功能

#### 📊 查看实时数据

连接成功后，您应该能看到：
- 实时图表显示角度、速度、电压数据
- 传感器数据面板显示当前数值
- 通信日志显示接收到的数据包

#### ⚙️ 调试PID参数

1. **获取当前PID**
   ```
   点击 "📥 获取PID" 按钮
   → 等待接收PID数据
   → 输入框会自动填充当前参数
   ```

2. **修改并发送PID**
   ```
   在输入框中修改参数值
   → 点击 "📤 发送PID" 按钮
   → 等待发送完成提示
   ```

3. **保存PID到EEPROM**
   ```
   点击 "💾 保存到EEPROM" 按钮
   → 参数将永久保存
   → 下次开机时自动加载（需按住按键启动）
   ```

#### 🕹️ 控制小车运动

使用方向控制按钮：
- ⬆️ **前进** - 小车向前移动
- ⬇️ **后退** - 小车向后移动
- ⬅️ **左转** - 小车向左转向
- ➡️ **右转** - 小车向右转向
- ⏹️ **停止** - 停止所有运动

**注意**: 确保小车有足够的运动空间，避免跌落或碰撞！

## 📖 常见使用场景

### 场景1：初次调试PID参数

```
1. 串口连接Arduino
2. 点击"获取PID"查看当前参数
3. 观察角度图表，小车是否能保持平衡
4. 如果不平衡，微调Balance_Kp（通常10-20之间）
5. 调整Balance_Kd抑制震荡（通常0.1-1.0之间）
6. 满意后点击"保存到EEPROM"
```

### 场景2：速度环调试

```
1. 确保平衡环参数已调好
2. 观察速度图表
3. 调整Velocity_Kp控制速度响应（通常1-5之间）
4. 调整Velocity_Ki消除稳态误差（通常0.001-0.1之间）
5. 保存参数
```

### 场景3：无线遥控

```
1. 蓝牙连接小车
2. 使用运动控制按钮
3. 观察传感器数据和图表
4. 实时监控小车状态
```

## ⚠️ 注意事项

### 重要提醒

1. **小车状态**
   - 获取/设置PID需要小车在运行状态（未按停止按钮）
   - 停止状态下Arduino不会响应PID命令

2. **参数调整**
   - 每次只调整一个参数
   - 调整幅度不要太大（建议每次±10%）
   - 先调平衡环，再调速度环

3. **安全第一**
   - 调试时将小车放在安全的平面上
   - 避免小车跌落导致损坏
   - 电压低于10.5V时及时充电

## 🔧 故障排除

### 问题1：找不到蓝牙设备
**解决方法**:
- 检查蓝牙模块是否上电（LED应该闪烁）
- 确保蓝牙未被其他设备连接
- 重启蓝牙模块
- 使用串口连接替代

### 问题2：串口连接失败
**解决方法**:
- 关闭Arduino IDE和其他串口工具
- 检查USB线连接
- 尝试不同的波特率
- Windows用户检查设备管理器中的COM口

### 问题3：PID参数无法获取
**解决方法**:
- 确保小车处于运行状态
- 检查通信日志查看是否收到数据
- 尝试断开重连
- 检查Arduino代码是否正确

### 问题4：数据刷新很慢
**解决方法**:
- 检查Arduino发送频率（建议50-100ms）
- 降低图表最大数据点数（在config.js中修改）
- 关闭浏览器其他标签页

### 问题5：控制命令不响应
**解决方法**:
- 检查通信日志确认命令已发送
- 确认Arduino代码中有对应的命令处理
- 检查小车是否处于可控制状态

## 📚 进阶使用

### 自定义配置

编辑 `js/config.js` 文件可以修改：
- 图表显示样式和数据点数量
- 波特率选项
- 电压警告阈值
- UI主题和行为
- 调试选项

### 修改协议

如果您的Arduino使用不同的通信协议：
1. 修改 `js/protocol.js` 中的正则表达式
2. 添加对应的解析逻辑
3. 更新命令生成函数

### 添加新功能

模块化设计使得扩展功能非常简单：
1. 在对应模块文件中添加方法
2. 在 `app.js` 中调用新方法
3. 在 `index.html` 中添加UI控件

## 💡 最佳实践

1. **调试前**
   - 确保硬件连接正常
   - 检查电池电量充足
   - 在安全环境中测试

2. **调试中**
   - 保存每次成功的参数
   - 记录调整过程和结果
   - 观察图表趋势而非瞬时值

3. **调试后**
   - 保存最终参数到EEPROM
   - 备份参数值
   - 记录在README中
