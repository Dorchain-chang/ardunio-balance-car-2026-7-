# 平衡小车PID调试上位机使用说明

## 概述

本文档介绍了为平衡小车项目开发的网页版PID调试上位机的功能、使用方法和技术实现。该上位机通过Web Serial API与Arduino通信，实时显示小车状态数据，并允许用户动态调整PID参数。

## 功能特性

1. **实时数据显示**：通过图表实时展示小车的角度、速度、电压等关键数据
2. **PID参数调整**：支持在线调整平衡环和速度环的PID参数
3. **参数保存**：可以将调试好的PID参数保存到Arduino的EEPROM中
4. **数据可视化**：使用Chart.js绘制直观的实时曲线图
5. **跨平台兼容**：支持所有支持Web Serial API的现代浏览器

## 文件结构

- `pid_tuner_web.html`：网页版PID调试上位机的主文件
- `Minibalance_for_Arduino_Final.ino`：已修改的Arduino主程序，支持与上位机通信

## 技术实现

### 1. Web Serial API通信

网页通过Web Serial API与Arduino建立串口通信，支持双向数据传输：

- **数据接收**：解析Arduino发送的格式如`{A:left:right:voltage:angle}$`或`{B:angle:voltage:left:right}$`的数据
- **命令发送**：发送PID参数设置、获取和保存命令

### 2. 实时数据可视化

使用Chart.js库绘制四个实时更新的图表：

- **角度图表**：显示小车的平衡角度变化
- **速度图表**：显示左右轮速度变化
- **PWM图表**：显示电机PWM输出
- **电压图表**：显示电池电压变化

### 3. Arduino端命令处理

在Arduino代码中添加了`processSerialCommands()`函数，用于：

- 解析来自网页的命令
- 实时更新PID参数
- 响应参数查询请求
- 执行EEPROM保存操作

## 使用方法

### 1. 准备工作

1. 将修改后的`Minibalance_for_Arduino_Final.ino`上传到Arduino开发板
2. 确保Arduino与电机驱动、MPU6050等硬件连接正确
3. 使用支持Web Serial API的浏览器（Chrome、Edge或Opera的最新版本）

### 2. 连接与初始化

1. 打开`pid_tuner_web.html`文件
2. 点击"连接串口"按钮，选择Arduino对应的串口设备
3. 连接成功后，页面会开始接收并显示小车数据

### 3. PID参数调试

1. 在网页的PID参数输入区域调整参数值：
   - **平衡环**：调整Kp和Kd参数，影响小车的平衡稳定性
   - **速度环**：调整Kp和Ki参数，影响小车的速度控制

2. 点击"发送PID参数"按钮，将新参数发送到Arduino
3. 观察图表上的数据变化，评估参数调整效果
4. 重复调整直到获得满意的平衡效果

### 4. 参数保存

1. 调试完成后，点击"保存PID到EEPROM"按钮
2. Arduino会将当前PID参数写入EEPROM，下次重启时自动加载

### 5. 获取当前参数

- 点击"获取PID参数"按钮，可从Arduino读取当前使用的PID参数
- 网页会自动更新输入框中的数值

## 通信协议说明

### Arduino发送的数据格式

1. **数据帧A**：`{A:left:right:voltage:angle}$`
   - left：左轮速度
   - right：右轮速度
   - voltage：电池电压（处理后的值）
   - angle：平衡角度

2. **数据帧B**：`{B:angle:voltage:left:right}$`
   - 与数据帧A包含相同信息，但顺序不同

3. **PID参数帧**：`{C:kp:kd:kp:ki}$`
   - 前两个参数：平衡环的Kp和Kd
   - 后两个参数：速度环的Kp和Ki
   - 所有参数都乘以100以整数形式传输

### 网页发送的命令格式

1. **设置PID参数**：`Ckp1:kd1:kp2:ki2$`
   - kp1, kd1：平衡环的Kp和Kd（整数形式，原数×100）
   - kp2, ki2：速度环的Kp和Ki（整数形式，原数×100）

2. **请求PID参数**：`P$`
   - 触发Arduino发送当前PID参数

3. **保存到EEPROM**：`F$`
   - 命令Arduino将当前PID参数保存到EEPROM

## 调试技巧

1. **平衡环调试**：
   - 先调大Kp使小车能够站立，再微调Kd减小震荡
   - 如果小车左右摇晃，可能需要调整机械中值（ZHONGZHI）

2. **速度环调试**：
   - 先设置较小的Kp，避免小车反应过度
   - 根据需要逐步增加Ki以消除稳态误差

3. **观察数据**：
   - 角度曲线应保持在0度附近小幅波动
   - 速度曲线应根据控制指令平滑变化
   - 电池电压低于10V时，小车会自动停止以保护电池

## 注意事项

1. 使用Web Serial API需要浏览器支持，建议使用Chrome最新版本
2. 确保串口连接稳定，避免调试过程中断开连接
3. 参数调整应循序渐进，避免大幅更改导致小车失控
4. 保存到EEPROM的操作次数有限（约10万次），请合理使用

## 性能优化建议

1. **通信优化**：
   - 可根据实际需要调整数据发送频率，减少通信开销
   - 考虑使用二进制通信代替ASCII，提高数据传输效率

2. **计算优化**：
   - 在Arduino端可以优化PID计算，减少CPU占用
   - 考虑使用更高效的滤波算法

3. **扩展性提升**：
   - 可以添加更多数据通道，监控电机电流、温度等参数
   - 实现参数自动优化功能，通过算法寻找最佳PID参数

## 故障排除

1. **无法连接串口**：
   - 检查浏览器是否支持Web Serial API
   - 确认Arduino驱动已正确安装
   - 尝试关闭其他可能占用串口的程序

2. **数据显示异常**：
   - 检查波特率设置是否正确（应为9600）
   - 确认Arduino代码已正确上传
   - 观察原始数据输出，检查格式是否匹配

3. **参数调整无响应**：
   - 确保命令格式正确
   - 检查Arduino串口缓冲区是否溢出
   - 尝试重置Arduino后重新连接
